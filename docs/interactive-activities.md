# Интерактив для LP Crew → рост конверсии в заявку

Документ-анализ: какие интерактивные активности (про плавание) добавить на сайт, чтобы пользователь вовлёкся и в финале захотел записаться на тренировку.

---

## 1. Контекст и ограничения

| Параметр | Текущее состояние |
|---|---|
| Стек | Django 5.1 + Wagtail 6.3, server-rendered шаблоны |
| Фронт | Чистый vanilla JS (нет React/Vue/Alpine), монолитный `style.css`, тема NOIR (тёмный фон, акцент cyan `#8fd6e0`) |
| Язык | Русский, аудитория ХМАО + онлайн |
| Точка конверсии | Секция `#join` → форма `name / phone / messenger` → `/leads/submit/` → Telegram-уведомление тренеру |
| Модель лида | `Lead(name, phone, messenger, created_at, handled)` |
| Контент | 22 водоёма, 380 км открытой воды, 6+ заплывов, 3 тренера, дисциплины: бассейн / открытая вода / онлайн / разбор техники |

**Главный вывод:** сайт — красивый лендинг-витрина, но интерактив сейчас декоративный (счётчики, карусели, аккордеон FAQ). Ни один элемент **не ведёт целенаправленно в форму**. Любая новая активность должна заканчиваться в `#join` — иначе это украшение, не конверсия.

**Принцип отбора:** активность хороша, если (а) про плавание, (б) даёт пользователю ценность за 30–90 сек, (в) её результат логично продолжается фразой «а теперь запишись, и мы поможем».

## 2. Рекомендация: 3 активности по приоритету

### 🥇 Приоритет 1 — Квиз «Какой ты пловец?» (профайлер)

Короткий квиз 5–6 вопросов → персональный результат-тип пловца → **предзаполненная заявка** под его цель.

**Почему конвертит лучше всего:**
- Микро-вовлечение: первый клик дешёвый, дальше работает эффект завершения (sunk-cost — начал, хочется узнать результат).
- Результат = персонализация. Вместо обезличенного «запишись» — «Тебе подходит *Открытая вода, группа выходного дня*. Оставь телефон — подберём старт».
- Профиль пользователя прокидывается тренеру в заявку → менеджер звонит уже подготовленным → выше доходимость.
- Геймификация без приза: люди любят узнавать «свой тип».

**Логика вопросов (пример):**
1. Уровень: не умею / держусь на воде / плаваю кролем / опытный
2. Цель: похудеть / техника / марафон-открытая вода / соревнования / для здоровья
3. Где удобно: бассейн ХМАО / открытая вода летом / онлайн из любого города
4. Как часто: 1 / 2–3 / 4+ раз в неделю
5. Дистанция мечты: 500 м / 1 км / 5 км / переплыть водоём
6. (опц.) Возрастная группа / есть ли страх глубокой воды

**Маппинг результата → дисциплина LP Crew:**

| Профиль | Рекомендация (существующий продукт) |
|---|---|
| Новичок + здоровье | Бассейн, постановка техники с тренером |
| Уверенный + открытая вода | Летние заплывы, группа выходного дня |
| Любой + другой город | Онлайн-план тренировок |
| Опытный + соревнования | Индивидуальный разбор техники по видео |

**Финальный экран = CTA:**
> «Твой тип: *Покоритель открытой воды* 🌊
> Ближайший подходящий старт — заплыв на 1 км.
> Оставь телефон — забронируем место и пришлём план подготовки.»
> [ форма name + phone + messenger, уже знающая выбранную дисциплину ]

**Интеграция со стеком (минимальные изменения):**
- Новый Wagtail Stream-block `quiz` (вопросы/ответы редактируются в админке) → шаблон `home/blocks/quiz.html`.
- Логика — отдельный `static/js/quiz.js` (vanilla, как остальной фронт): шаги, прогресс-бар, подсчёт профиля. Без бэкенда — считается в браузере.
- Результат скроллит к `#join` и предзаполняет скрытое поле. Чтобы тренер видел профиль — расширить `Lead`:
  ```python
  # leads/models.py
  profile = models.CharField("Профиль из квиза", max_length=120, blank=True)
  goal    = models.CharField("Цель", max_length=120, blank=True)
  ```
  и добавить эти поля в `LeadForm.Meta.fields` + в текст Telegram-уведомления (`notify.py`).
- Анти-спам/ratelimit уже есть — переиспользуется `/leads/submit/`.

**Оценка:** ~1.5–2.5 дня. Самый высокий ROI.

### 🥈 Приоритет 2 — Калькулятор «Доплыву ли я?» (темп / время / дистанция)

Пользователь вводит дистанцию (или выбирает водоём из 22) и свой примерный темп → получает расчётное время, расход калорий и «уровень вызова». Под результатом — призыв подготовиться с тренером.

**Почему работает:**
- Чистая польза + лёгкая самопроверка → доверие к экспертизе команды.
- Привязка к **реальным водоёмам LP Crew** = бренд-контент, которого нет у конкурентов.
- Результат «5 км — это серьёзно, средний любитель плывёт ~2 ч» естественно ведёт к «подготовься с нами».

**Пример вывода:**
> Озеро X · 1 км · твой темп 2:30/100м
> ⏱ ~25 мин · 🔥 ~320 ккал · Уровень: средний
> «Готов к такой дистанции? Запишись на пробную — проверим технику.» → `#join`

**Интеграция:** один блок `calc` + `static/js/calc.js`, расчёт в браузере. Данные водоёмов можно брать из заплывов (`SwimPage.distance/place`). CTA → `#join` с пометкой «интересует: открытая вода».

**Оценка:** ~1–1.5 дня. Хорошо ставится в пару к квизу (для тех, кто уже знает свою цель).

### 🥉 Приоритет 3 — Интерактивная карта «22 водоёма»

Карта точек открытой воды, где команда плавала: клик по точке → фото из галереи заплыва, дистанция, история. Внизу — «Хочешь в следующий заплыв? Запишись».

**Почему ниже приоритетом:**
- Сильный «вау» и удержание на странице, **но слабее конвертит** — это исследование, не действие. Работает на доверие/бренд, не на прямой лид.
- Дороже: нужен картографический слой (Leaflet + OSM, без тяжёлых SDK) и геоданные по 22 точкам.

**Когда делать:** после квиза и калькулятора, как контент-усилитель раздела заплывов. Переиспользует `SwimPage` (place, cover, gallery, distance).

**Оценка:** ~2–3 дня (в основном — сбор координат и привязка к заплывам).

## 4. Что внедрить в первую очередь

**Квиз** — единственная активность, которая одновременно: вовлекает, персонализирует и **приносит обогащённый лид** (тренер видит уровень+цель ещё до звонка). Это прямой рычаг доходимости заявок, не просто украшение.

Минимальный путь к запуску:
1. Расширить `Lead` полями `profile`, `goal` + миграция.
2. Wagtail-блок `quiz` (вопросы редактируются контентом) + `quiz.js`.
3. Финальный экран скроллит в `#join`, прокидывает профиль в скрытые поля.
4. `notify.py` шлёт тренеру профиль вместе с контактом.

Калькулятор — быстрый второй шаг (переиспользует те же `#join` и данные заплывов). Карта — поздний бренд-усилитель.

## 4.5. Игра (геймификация)

Да, игру добавить можно и стоит — она даёт удержание и виральность («скинь другу, обгони меня»), чего нет у квиза/калькулятора. **Но то же правило:** конец игры = заявка. Иначе это игра ради игры, не конверсия. Рычаг — **таблица лидеров**: чтобы попасть в топ, нужно оставить контакт. Так результат игры превращается в лид.

Два варианта — дешёвый и крупный.

### 🎮 Вариант A (рекомендую начать) — «Задержи дыхание»

Челлендж-таймер: жми и держи кнопку, пока можешь «задержать дыхание». Отпустил → твоё время. Сравнение с уровнями.

**Почему хорош:**
- Прямая ассоциация с плаванием (апноэ — база для открытой воды).
- Очень дёшево: таймер + одна кнопка, ~полдня. Никакого движка, физики, ассетов.
- Виральный крючок: «держу 45 сек, а ты?».
- Естественный CTA: «Наши пловцы держат 2+ минуты. Хочешь так же? Дыхательные техники — на тренировке.»

**Шкала результата:**
| Время | Реакция |
|---|---|
| < 30 сек | «Новичок — научим дышать правильно» |
| 30–60 сек | «Неплохо! С тренером — больше» |
| 60–120 сек | «Сильно. Тебе к открытой воде» |
| 2+ мин | «Уровень LP Crew 🌊 Запишись на заплыв» |

Финал → `#join` с пометкой «прошёл челлендж дыхания».

**Оценка:** ~0.5–1 день.

### 🎮 Вариант B — «Заплыв» (аркада, endless swimmer)

Бесконечный мини-раннер: пловец плывёт, уворачивается от препятствий (буйки, лодки, медузы), счётчик метров растёт. Game over → дистанция + таблица лидеров.

**Почему сильный:**
- Счётчик метров бьётся с брендом («380 км открытой воды» в stats) — «ты проплыл 540 м виртуально, проплыви по-настоящему».
- Таблица лидеров = механизм сбора лидов: «войди в топ-10 — оставь имя и контакт».
- Возвращаемость: люди перезаходят бить рекорд.

**CTA на Game Over:**
> «Дистанция: 540 м 🏊 Место в рейтинге: #7
> Хочешь проплыть это в реальной воде? Оставь контакт — попадёшь в таблицу и позовём на старт.»
> → форма `#join`

**Интеграция со стеком:**
- Vanilla JS на `<canvas>` (`static/js/game.js`), в духе остального фронта — без игровых движков.
- Таблица лидеров = новая лёгкая модель `Score(name, meters, created_at)` ИЛИ переиспользовать заявку: топ показываем из лидов с заполненным `meters`.
- Отправка рекорда идёт через тот же `/leads/submit/` (ratelimit/honeypot уже есть) + поле `meters` в `Lead`.
- Тема NOIR ложится идеально: тёмный фон, акцент cyan, силуэт пловца.

**Риск:** дороже и может «съесть» внимание вместо заявки, если рейтинг не привязать к контакту. Привязка обязательна.

**Оценка:** ~3–4 дня.

### Куда вписать

- «Задержи дыхание» — лёгкий блок рядом с дисциплинами или в FAQ-зоне, как «разминка».
- «Заплыв» — отдельная секция-аттракцион или мини-страница `/game/`, ссылка из hero/меню.

| Игра | Конверсия | Усилие | Виральность | Старт |

| Задержи дыхание | 🟢 высокая | очень низкое | высокая | **первой** |
| Заплыв (аркада) | 🟡 средн.→высокая (с топом) | высокое | очень высокая | позже |

## 5. Метрики успеха

- CR старта квиза → завершения (целевой ≥ 50%).
- CR завершения квиза → отправки заявки (целевой ≥ 20%).
- Доля заявок из квиза в общем потоке `/leads/submit/`.
- Доходимость лида до тренировки: квиз-лиды vs обычные (ожидаемо выше за счёт персонализации).
